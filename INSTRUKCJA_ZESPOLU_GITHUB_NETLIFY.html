Wspolna praca grafikow: GitHub + Cursor + Netlify

Szybka instrukcja dla zespolu: GitHub + Cursor + Netlify.

Cel: kazdy grafik pracuje na tych samych plikach, a podglad online aktualizuje sie automatycznie po kazdym git push.

1) Pierwsze uruchomienie (jednorazowo)

Linki startowe:
  • GitHub (konto / logowanie): https://github.com/
  • Cursor (pobranie): https://www.cursor.com/
  • Git (instalacja): https://git-scm.com/downloads
  • Netlify (konto): https://www.netlify.com/
  1. Zaloz konto na github.com (plan Free).
  2. Zainstaluj Git i Cursor.
  3. W Cursor otworz terminal: View -> Terminal (lub Terminal -> New Terminal).
Uwaga o skrocie: to jest Ctrl + ` (backtick), nie Ctrl + '.

2) Wrzucenie nowego projektu na GitHub (owner projektu)

A. Ustaw dane Git (raz na komputerze)

git config --global user.name "Imie Nazwisko"
git config --global user.email "twoj@email.com"
Co jest czym i gdzie to znalezc:
  • user.name to podpis autora commitow - mozna wpisac dowolna czytelna nazwe, np. "Jan Kowalski" albo "Studio Pixel".
  • user.email to email autora commitow - najlepiej taki sam jak email konta GitHub.
  • GitHub username (login) to nazwa konta w adresie profilu, np. https://github.com/twoj-login.
  • Ten login widac tez po zalogowaniu, w menu konta w prawym gornym rogu GitHub.
  • Login GitHub uzywasz w adresie repo, a user.name sluzy tylko jako podpis commitow.

B. Przejdz do folderu projektu

Windows (zalecane):

Set-Location "D:\Sciezka\Do\Twojego-Projektu"

Alternatywnie:

cd "D:\Sciezka\Do\Twojego-Projektu"

C. Pierwszy commit

git init
git add .
git commit -m "Pierwszy commit"

Tekst "Pierwszy commit" moze zostac dokladnie taki sam. To tylko opis pierwszego zapisu projektu.

D. Utworz puste repo na GitHubie

  • New repository - nazwa np. projekt-graficzny-klient-a
  • Visibility: Private

E. Podepnij i wypchnij projekt

git remote add origin https://github.com/TWOJ_LOGIN/NAZWA_REPO.git
git branch -M main
git push -u origin main

3) Podglad online w Netlify

  1. Zaloguj sie do netlify.com przez GitHub.
  2. Add new site -> Import an existing project.
  3. Wybierz swoje repo projektu.
  4. Kliknij Deploy site.

Dla React/Vite najczesciej ustaw: Build command: npm run build, Publish directory: dist.

Od teraz: kazdy push na GitHub = automatyczna aktualizacja podgladu.

4) Dodanie grafikow do projektu

  1. GitHub repo -> Settings -> Collaborators -> Add people.
  2. Kazdy grafik akceptuje zaproszenie.
  3. Kazdy grafik klonuje repo (jednorazowo):
git clone https://github.com/TWOJ_LOGIN/NAZWA_REPO.git

5) Gdy repo juz istnieje i chcesz tylko dzialac na projekcie online

Ten scenariusz jest dla grafikow, ktorzy dolaczaja do juz gotowego projektu zespolowego.
  1. Popros ownera o dodanie Ciebie jako collaborator w GitHub.
  2. Zaakceptuj zaproszenie z maila lub z powiadomienia na GitHub.
  3. W Cursor otworz terminal i sklonuj repo:
git clone https://github.com/TWOJ_LOGIN/NAZWA_REPO.git
cd "NAZWA_REPO"
  1. Uruchom projekt lokalnie zgodnie z README (np. npm install, potem npm run dev).
  2. Pracuj na branchu/glownej galezi zgodnie z zasadami zespolu i wysylaj zmiany przez git push.

6) Codzienny rytm pracy

Przed praca: 
git pull
Po pracy: 
git add .
git commit -m "Krotki opis zmian"
git push

Alternatywnie mozna poprosic Cursora po polsku o wykonanie tych krokow.

7) Gdy pojawi sie konflikt

  1. Nie panikuj - to normalne przy pracy zespolowej.
  2. Popros Cursora: Mam konflikt w Gicie, pomoz mi go rozwiazac.
  3. Po rozwiazaniu konfliktu:
git add .
git commit -m "Resolve merge conflict"
git push

Najwazniejsze: nie usuwaj plikow "na sile", tylko rozwiazuj konflikt swiadomie.

Dokument zespolowy. W razie pytan: wrzuc screen bledu i tresc komendy z terminala.
